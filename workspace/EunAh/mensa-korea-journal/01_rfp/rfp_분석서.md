# RFP 분석서 — 멘사코리아 회지 제작 자동화 시스템

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 멘사코리아 회지 제작 자동화 시스템 |
| 클라이언트 | 멘사코리아 |
| 핵심 목표 | "문서를 받는 것이 아니라, 데이터를 받는다" — 원고 입력 단계부터 구조화 → 교정·태그 관리 → 디자이너 전달까지 단일 워크플로우 통합 |
| 구축 방식 | 웹 기반 CMS (내부 사용 시스템, 대외 서비스 아님) |
| 사용 대상 | 담당자(관리자) 1인 + 다수 원고 작성자 |
| 기술 스택 | PHP / MySQL / Velzon 4.4.1 / Quill.js / PHPWord |

## 4. 스타일 태그 정의

| 기호 | 태그명 | 적용 규칙 | 사진 채번 여부 |
|------|--------|-----------|--------------|
| ♣ | 사진 | 사진 삽입 위치 표시 | ♣01, ♣02... 자동 채번 |
| ♥ | 소제목 강조 | 소제목 강조 처리 | - |
| ★ | 질문 | 질문 또는 인터뷰 질문 | - |
| ● | 진하게 | 굵게(볼드) 처리 | - |
| ■ | 본문 내 강조 A | 다른 폰트 + 다른 색 + 굵게 | - |
| ◈ | 본문 내 강조 B | 같은 폰트, 굵게만 | - |

**채번 규칙**: 전체 원고 순서 기준 ♣01, ♣02... 자동 부여. 원고 순서 변경 시 자동 재정렬.

## 5. 리스크 포인트

| 리스크 | 설명 | 영향도 | 대응 방안 |
|--------|------|--------|----------|
| hwp 파싱 복잡도 | PHP에서 hwp 직접 파싱 어려움 (독점 포맷) | 높음 | hwp2html 변환 도구 활용, 또는 hwp 지원 범위 제한 |
| Quill.js 커스텀 태그 | ♣♥★●■◈ 기호를 에디터 내 특수 처리 필요 | 중간 | Quill Blot 커스텀 구현 또는 텍스트 삽입 방식 |
| docx Export 서식 매핑 | 태그→Word 서식 변환 매핑 복잡 (PHPWord) | 중간 | 태그별 Word 스타일 매핑 표 사전 정의 필요 |
| 사진 번호 재정렬 | 원고 순서 변경 시 전체 ♣ 번호 재채번 | 중간 | DB에 사진 위치 순서 저장, 순서 변경 시 트리거 처리 |
| 파일 파싱 보안 | 악성 파일 업로드 위험 | 높음 | MIME 타입·확장자 검사, 업로드 전용 디렉토리 격리 |
