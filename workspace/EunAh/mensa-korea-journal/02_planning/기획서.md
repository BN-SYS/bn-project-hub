# 멘사코리아 회지 제작 자동화 시스템 — 기획서 v1.0

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 멘사코리아 회지 제작 자동화 시스템 |
| 클라이언트 | 멘사코리아 |
| 구축 목표 | 원고 수신 → 서식 정제 → 교정 → 태그 → 사진 채번 → 디자이너 전달 단일 워크플로우 |
| 사용자 | 담당자(관리자) 1인 + 원고 작성자 다수 |
| 개발 방식 | 웹 기반 내부 CMS (사내 사용, 대외 서비스 아님) |
| 기술 스택 | PHP MVC / MySQL / Velzon 4.4.1 (관리자 UI) / Quill.js / PHPWord |

---

## 2. 시스템 구성

### 2.1 아키텍처

```
[작성자]  →  원고 입력 폼 (웹)
                ↓ 제출
[담당자]  →  원고 취합 목록 (Velzon 관리자 UI)
                ↓
             원고 편집 + 태그 삽입 (Quill.js 에디터)
                ↓ 순서 확정
             원고 순서 관리 (드래그)
                ↓ 사진 자동 채번
             이미지 관리 (번호 매핑 확인)
                ↓
             Export → .docx 다운로드
                ↓
[디자이너]  ← 최종 취합본 + 이미지 파일 수령
```

### 2.2 기술 구성

| 레이어 | 기술 | 역할 |
|--------|------|------|
| Frontend (관리자) | Velzon 4.4.1 + Quill.js | 관리자 대시보드·편집 UI |
| Frontend (작성자) | Velzon 스타일 기반 심플 폼 | 원고 입력 전용 |
| Backend | PHP MVC | 라우팅·컨트롤러·모델 |
| Database | MySQL | 원고·이미지·회지·사용자 데이터 |
| 파일 파싱 | PHPWord (docx), txt 직접 처리 | 원고 파일 텍스트 추출 |
| 에디터 | Quill.js (커스텀 블롯) | 태그 삽입 UI |
| Export | PHPWord | .docx 생성 |
| 순서 관리 | SortableJS | 드래그 앤 드롭 |

## 3. 사이트맵

### 3.1 작성자 영역 (심플 레이아웃)

```
/ (로그인)
└── /my/articles          ← 내 원고 목록
    └── /my/articles/new  ← 원고 입력 폼 (핵심 화면)
    └── /my/articles/{id} ← 제출 완료 확인
```

### 3.2 관리자 영역 (Velzon 레이아웃)

```
/admin (로그인)
└── /admin/dashboard              ← 현재 회지 진행 현황
└── /admin/journals               ← 회지 호수 목록
    └── /admin/journals/{id}      ← 회지 상세 (= 원고 목록)
└── /admin/articles               ← 전체 원고 목록
    └── /admin/articles/{id}/edit ← 원고 편집 (Quill.js + 태그 UI)
└── /admin/articles/order         ← 원고 순서 관리 (드래그)
└── /admin/images                 ← 이미지 관리 (채번 확인)
└── /admin/export                 ← Export (.docx 다운로드)
└── /admin/users                  ← 작성자 계정 관리
```

## 5. 주요 기능 정의

### 5.1 원고 입력 (Phase 1)
- 웹 폼: 제목·작성자명·본문(Quill.js)·이미지·캡션 구조화 입력
- 파일 업로드: docx / txt 파일 → 서버에서 텍스트 추출 → 에디터에 자동 삽입
  - ⚠️ hwp 파싱은 별도 확인 필요 (Q1)
- 서식 자동 정제: 불필요 공백·줄바꿈 정규화 (서버사이드 자동 처리)

### 5.2 스타일 태그 삽입 (Phase 1)
- Quill.js 툴바에 커스텀 버튼 6개 추가
- 버튼 클릭 시 커서 위치에 기호 삽입 (♣ ♥ ★ ● ■ ◈)
- ♣ 삽입 시 자동 채번 처리 (현재 원고 내 순번)

### 5.3 원고 순서 관리 및 사진 채번 (Phase 1)
- SortableJS로 원고 순서 드래그 조정
- 순서 저장 시 전체 원고의 ♣ 번호 자동 재채번 (♣01, ♣02...)
- 번호는 DB에 저장, 에디터 표시에 실시간 반영

### 5.4 이미지 관리 (Phase 1)
- 원고 제출 시 이미지 첨부 → 채번된 번호로 자동 파일명 매핑
- 관리자: 이미지 목록에서 번호-파일명-원고 연결 확인
- Export 시 이미지 별도 폴더 제공 (기본값, Q4 확인)

### 5.5 워드 Export (Phase 1)
- 취합 순서대로 전체 원고 합본
- 스타일 태그 → Word 스타일 매핑 적용
  - ♥ → 소제목 스타일 (제목2)
  - ★ → 인터뷰 질문 스타일 (강조)
  - ● → Bold
  - ■ → 다른 폰트+색+Bold
  - ◈ → Bold만
  - ♣01 → 사진 자리표시 텍스트
- PHPWord로 .docx 생성 후 다운로드

### 5.6 변경 추적·코멘트·버전 관리 (Phase 2)
- diff-match-patch로 수정 내역 시각화
- 문장 단위 코멘트 작성
- 버전별 저장·복원

### 5.7 InDesign 연동 (Phase 3)
- XML / JSON Export (태그·이미지 경로·캡션 포함)
