결정 사항 1 · 투표자 인증 방식 및 모수 인정 기준 확인 필요

인증 방식 (현행 유지): 이름 + 전화번호 + 주문번호 끝 5자리를 그대로 유지합니다. 단, 인증 통과 여부와 별도로 기준인원 명단 포함 여부를 추가로 확인합니다.

현행 전제 조건: 투표에 직접 참여하려면 쇼핑몰 회원가입 + 티켓 구매가 선행되어야 합니다. 위임장을 제출한 비구매자는 직접 인증 없이, 수임인(티켓 구매자)이 대신 투표를 행사합니다.
기준인원 명단 매칭 기준: 이름 + 회원번호 (전화번호는 변경될 수 있어 제외)
상황처리
인증 통과 + 회원번호가 기준인원 명단에 존재✅ 투표 가능, 모수에 포함
인증 통과 + 회원번호가 기준인원 명단에 없음❌ 투표 화면 진입 불가, 모수 제외

향후 로드맵 (제안): 정회원 사이트 개발이 완료되는 시점에 맞춰, 내년 총회부터는 정회원 로그인 인증으로 전환하는 방식을 제안드립니다.

  2. 향후에는 정회원 로그인 인증으로 전환하는 로드맵에 동의하실까요?

01 안건 상태 추가 — "준비중" · "제외"
현재 상황

현재 안건 상태는 열림 ↔ 닫힘 두 단계만 있습니다. 강제 제외 시 정상 마감과 구분이 불가하며, 제외 안건도 투표자 화면 카운트에 그대로 포함되어 혼란이 발생합니다. 또한 사전에 안건 등록 후 원하는 시점에 투표 시작이 불가능합니다.

변경 내용
상태의미구분
준비중안건 등록 직후 기본 상태. 투표 미시작.신규
열림현재 투표 진행 중.현행 유지
닫힘정상 마감. 결과 집계에 포함. 미투표자 자동 처리.현행 유지
제외강제 제외. 결과 집계에서 제외.신규

상태 전환 흐름:

준비중 → 열림 ⇄ 닫힘
준비중 → 제외 가능  |  열림 → 제외 가능  |  닫힘 이후 제외 불가

제외 처리 규칙: 준비중·열림 상태에서만 가능, 2단계 확인 팝업 필수, 제외 후 재포함 불가, 기존 투표 데이터는 내부 보존.

10 의장 일괄 투표 — 선택지별 분할 투표 행사
현재 상황

의장에게 귀속된 위임 건들을 투표 행사할 때, 안건당 하나의 선택지를 골라 전체 위임자에게 일괄 적용하는 방식입니다.

변경 내용

각 선택지에 몇 %씩 분배할지 입력하여 위임 인원을 선택지별로 나눠 투표할 수 있게 합니다.

모드설명
명수 직접 입력각 선택지에 배분할 인원 수를 직접 숫자로 입력
% 입력각 선택지에 배분할 비율(%)을 입력하면 자동으로 명수 계산

입력 규칙: 입력한 값의 합계가 의장 귀속 총 인원 수와 일치해야 제출 가능. % 모드에서 소수점 반올림 발생 시 차이 인원은 가장 높은 비율의 선택지에 자동 배분.

bulk-vote.php 화면 예시 (안건당)

[ 안건 1. 정관 개정안 승인의 건 ] — 의장 귀속: 10명

선택지비율 (%)→ 명수
찬성60%6명
반대30%3명
기권10%1명
합계 100% ✅10명
⚠ 확인 필요
  • 의장 귀속 위임 인원을 선택지별로 나눌 때, 각 위임자를 어떤 선택지에 배정할지는 시스템이 임의로 결정합니다. (예: 찬성 6명 배정 시 명단 상위 6명 → 찬성으로 기록) 이 방식이 의결 효력상 문제가 없는지 확인 부탁드립니다.
