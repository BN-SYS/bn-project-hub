### 결정 사항 1 · 투표자 인증 방식 및 모수 인정 기준 — ⚠️ 확인 필요

**제가 이해한 방향이 맞는지 확인 부탁드립니다.**

**인증 방식 (현행 유지):**

현재 인증 방식(이름 + 전화번호 + 주문번호 끝 5자리)을 그대로 유지합니다.  
단, 인증 통과 여부와 별도로 **기준인원 명단 포함 여부를 추가로 확인**합니다.

> 현행 방식의 전제 조건: 투표에 직접 참여하려면 **쇼핑몰 회원가입 + 티켓 구매**가 선행되어야 합니다.  
> 위임장을 제출한 비구매자는 직접 인증 없이, 수임인(티켓 구매자)이 대신 투표를 행사합니다.

기준인원 명단 매칭 기준은 **이름 + 회원번호**입니다. (전화번호는 변경될 수 있어 제외)

| 상황 | 처리 |
|---|---|
| 인증 통과 + 회원번호가 기준인원 명단에 존재 | 투표 가능, 모수에 포함 |
| 인증 통과 + 회원번호가 기준인원 명단에 없음 | 투표 화면 진입 불가, 모수 제외 |

**향후 로드맵 (제안):**

정회원 사이트 개발이 완료되는 시점에 맞춰, **내년 총회부터는 정회원 로그인 인증으로 전환**하는 방식을 제안드립니다.  
(정회원 로그인 = 이름·회원번호·전화번호 조합보다 강한 본인 인증 수단으로 보완)

> **확인 필요**  
> 1. 이번에는 현재 주문번호 인증을 유지하되, 기준인원 명단 존재 여부를 추가 확인하는 방식으로 진행을 원하시는 게 맞을까요?  
> 2. 향후에는 정회원 로그인 인증으로 전환하는 로드맵에 동의하실까요?

### 결정 사항 2 · 득표율 계산 시 분모(모수) 기준 — ⚠️ 확인 필요

**제가 이해한 모수 산정 방식이 맞는지 확인 부탁드립니다.**

**모수에 포함되는 기준:**

기준인원 명단 매칭은 **이름 + 회원번호** 기준입니다.

| 조건 | 포함 여부 |
|---|---|
| 쇼핑몰 티켓 구매자 중 회원번호가 기준인원 명단에 있는 경우 | ✅ 포함 |
| 위임인의 이름+회원번호가 기준인원 명단에 있고, 수임인의 이름+회원번호도 명단에 있는 경우 (위임인) | ✅ 포함 |
| 의장 위임 (수임인 미지정) — 위임인의 이름+회원번호가 기준인원 명단에 있는 경우 (위임인) | ✅ 포함 |
| 위 ✅ 조건 중 둘 이상 동시에 해당하는 동일인 (예: 티켓 구매자인 동시에 타인에게 위임한 경우) | ✅ 1명으로 카운트 |
| 티켓을 구매했으나 회원번호가 기준인원 명단에 없는 경우 | ❌ 제외 |
| 기준인원 명단에 있으나 티켓 미구매 + 위임 미제출 | ❌ 제외 |
| 위임인의 이름+회원번호가 기준인원 명단에 없는 경우 | ❌ 무효 위임 |
| 수임인의 이름+회원번호가 기준인원 명단에 없는 경우 (의장 위임 제외) | ❌ 무효 위임 |

**위임장 제출 시 처리 흐름:**

현재 투표 시스템에는 **티켓 구매자만 접근 가능**합니다. 비구매 위임장 제출자는 직접 접근할 수 없으며, 위임받은 수임인이 대신 행사합니다.

| 케이스 | 투표 행사 | 모수 포함 |
|---|---|---|
| 수임인이 티켓 구매자 + 현장 출석 | ✅ 수임인이 위임인 투표까지 함께 행사 | 위임인 + 수임인 모두 포함 |
| 수임인이 티켓 구매자이나 불참 | ❌ 투표 불가 | 위임인 + 수임인 모두 포함, 미투표 처리 |
| 수임인이 티켓 미구매자 (기준인원 명단에 있어도) | ❌ 투표 시스템 접근 불가 | 위임인 + 수임인 모두 포함, 미투표 처리 |
| 의장 위임 (수임인 미지정) | 의장(관리자)이 위임인 투표 행사 | 위임인 포함 |

**그 외 미투표 처리 케이스:**

- 티켓을 구매했지만 현장에 불참한 경우 → 모수 포함, 미투표

**위임 취소 및 투표권 복귀:**

위임장을 제출한 위임인의 티켓이 검표 처리되면 위임이 자동 취소되고, 투표권이 본인에게 복귀됩니다.  
현장 참석 여부는 구매한 티켓의 검표 처리로 확인합니다. 이는 다음 두 경우 모두 해당합니다.

- 비구매자로 위임장을 제출했으나 이후 직접 티켓을 구매하고 검표된 경우
- 티켓을 구매한 상태에서 위임장도 제출했으나 본인 티켓이 검표된 경우

위임 취소 후 투표 처리는 아래와 같습니다.

| 상황 | 처리 |
|---|---|
| 수임인이 아직 투표하지 않은 안건 (열림) | 위임인 본인이 직접 투표 |
| 수임인이 이미 투표한 안건 (열림) | 수임인 투표 데이터 유지. 위임인이 수정 가능 |
| 수임인이 이미 투표한 안건 (닫힘) | 수임인 투표 내용으로 확정. 수정 불가 |

> **확인 필요**  
> 위 모수 산정 기준 및 위임 처리 흐름이 운영 방침과 일치할까요?

### "기준인원 명단" 등록 기준

총회별 기준인원 명단을 아래 기준으로 등록합니다.

**등록 기준:**

- 대상: 해당 시점에 **멘사코리아 정회원 자격을 유지하는 회원**
- 제공 방식: 총회 전 관리자가 **Google 스프레드시트 파일의 기준인원 시트(회원명 / 회원번호 / 전화번호)** 업데이트

> **적용 범위:** 이미 종료된 총회는 제외. **개선 배포 이후 신규 등록 총회부터 적용.**

## 1. 안건 상태 추가 — "준비중" · "제외"

### 현재 상황

현재 안건 상태는 **열림 ↔ 닫힘** 두 단계만 있으며, 토글 방식으로 전환됩니다.  
총회 중 안건을 강제로 제외해야 할 때, 정상 마감(닫힘)과 구분할 방법이 없습니다.

이로 인해 **안건을 제외하거나 강제 종료한 경우에도 해당 안건이 투표자 화면의 안건 수에 그대로 카운트**됩니다.  
해당 안건에 투표하지 않은 투표자는 전체 진행현황에서 "미완료"로 표시되어 혼란을 겪고 있습니다.

또한 현재는 안건 등록 즉시 열림/닫힘 상태를 선택해야 하므로, **사전에 안건을 등록해두고 원하는 시점에 투표를 시작하는 것이 불가능**합니다.

### 변경 내용

**안건 상태에 "준비중"과 "제외" 항목이 추가됩니다.**

| 상태 | 의미 | 신규 여부 |
|---|---|---|
| **준비중** | 안건 등록 직후 기본 상태. 투표 미시작. | 신규 |
| 열림 | 현재 투표 진행 중. | 현행 유지 |
| 닫힘 | 정상 마감. 결과 집계에 포함. 미투표자 자동 처리. | 현행 유지 |
| **제외** | 강제 제외. 결과 집계에서 제외. | 신규 |

**상태 전환 흐름:**

```
준비중 → 열림 ⇄ 닫힘
   ↓       ↓
  제외    제외
```

- **열림 → 닫힘** 전환 시: 미투표자는 자동으로 미투표 처리
- **닫힘 → 열림 (재오픈)**: 허용. 재오픈 시 미투표 자동 처리가 초기화되며, 미투표자 재투표 가능. 기존 투표자도 투표 수정 가능.
- **제외**는 준비중·열림 상태에서만 가능 (닫힘 이후 불가)
- 제외 후 재포함 불가 (필요 시 안건 새로 등록)

**제외 처리 규칙:**
- 준비중 또는 열림 상태의 안건만 제외 가능
- 제외 시 **2단계 확인 팝업** 표시 (" 제외 시 복구가 불가합니다. 정말 제외하시겠습니까?" → 재확인)
- **제외 후 재포함 불가** — 필요 시 안건 새로 등록
- 제외된 안건의 투표 데이터는 삭제되지 않고 내부 보존

**각 화면에서의 제외 안건 처리:**

| 화면 | 처리 방식 |
|---|---|
| 관리자 총회 상세 | 안건 목록 **최하단으로 이동**, 순서 번호 제거. 회색 배경 + "제외" 배지 표시. 제외 전 기투표 건수(선택지별 표수)만 표시. 비율(%) 미표시. "부분 집계 (제외 처리됨)" 안내 표시. |
| 투표자 투표 화면 | **목록에 표시하지 않음** |
| 결과 CSV | **데이터에서 제외** |

> **개발 적용 범위**  
> - 작업 규모: 🔴 대 — DB 구조 변경 + 5개 이상 파일 동시 수정 필요  
> - 수정 파일: `assembly-detail.php` / `assembly-detail_agenda_onoff.php` / `vote.php` / `result.php` / `vote-status.php`  
> - DB 변경: `vote_assembly_info_agenda` 테이블의 `agenda_off` 상태값 체계 변경 (준비중·제외 추가)

## 2. 투표자 화면 — 진행현황 카운터 기준 변경

### 현재 상황의 문제점

현재 투표자 화면 상단의 진행현황 카운터("N개 중 M개 완료")에서 두 가지 문제가 있습니다.

1. **이미 마감된 안건에 투표를 못 한 경우** → "미완료"로 표시되어 투표자가 혼란을 겪음
2. **제외된 안건의 미투표자** → 마찬가지로 "미완료"로 잘못 표시됨

### 변경 내용

**진행현황 카운터 기준:**

| 항목 | 변경 후 기준 |
|---|---|
| 전체(N) | 준비중 + 열림 + 닫힘 안건 수. **제외 안건 미포함.** |
| 완료(M) | ① 실제 투표한 안건 + ② **마감(닫힘) 안건 전체 (투표 여부 무관)** |

준비중 안건은 전체 수에는 포함되나, 아직 투표가 시작되지 않았으므로 투표자 화면에서는 "대기 중"으로 표시하고 선택 불가 처리합니다.

마감된 안건은 더 이상 투표가 불가하므로, 투표를 못 한 경우 **미투표 처리**되며, 진행현황 카운터상으로는 **"완료"로 처리**합니다.

**안건별 상태 표시:**

| 상황 | 표시 |
|---|---|
| 투표 완료 | "완료" (초록) |
| 마감됨 + 미투표 | "마감" (회색) |
| 진행 중 + 미투표 | "미투표" (주황) |

**안내 문구 (조건부 표시):**  
마감된 안건 중 미투표 건이 있을 때 카운터 하단에 표시:
> "투표 마감 전 참여하지 못한 안건은 미투표 완료로 처리됩니다."

> **개발 적용 범위**  
> - 작업 규모: 🟡 중 — 항목 1(안건 상태 추가) 완료 후 진행 가능  
> - 수정 파일: `vote.php`

## (작업중) 3. 투표자 화면 — 안건 새로고침 버튼 추가

### 현재 상황

총회 진행 중 관리자가 새 안건을 추가해도, 투표자 화면에 자동으로 반영되지 않습니다.  
현재는 투표자가 브라우저 새로고침을 직접 해야 합니다(버튼 미제공).

### 변경 내용

투표자 화면 상단에 **"안건 새로고침" 버튼**이 추가됩니다.

- 아직 제출하지 않은 선택 항목이 있을 경우:  
  > "새로고침 시 미제출 선택이 초기화됩니다. 계속하시겠습니까?"  
  - 확인 → 새로고침  
  - 취소 → 현재 화면 유지
- 이미 제출 완료된 항목은 초기화 없이 새로고침

## 10. 의장 일괄 투표 — 선택지별 분할 투표 행사

### 현재 상황

현재 의장에게 귀속된 위임 건들을 투표 행사할 때, 안건당 **하나의 선택지를 골라 전체 위임자에게 일괄 적용**하는 방식입니다.  
(예: 의장 귀속 위임 10명 전원 → "찬성"으로 일괄 처리)

### 변경 내용

의장이 각 선택지에 **몇 %** 씩 분배할지 입력하여, 위임 인원을 **선택지별로 나눠 투표**할 수 있게 합니다.

**입력 방식 (전환 가능):**

| 모드 | 설명 |
|---|---|
| 명수 직접 입력 | 각 선택지에 배분할 인원 수를 직접 숫자로 입력 |
| % 입력 | 각 선택지에 배분할 비율(%)을 입력하면 자동으로 명수 계산 |

**입력 규칙:**

- 입력한 값의 합계가 **의장 귀속 총 인원 수와 일치**해야 제출 가능
- % 모드에서 소수점 반올림 발생 시, **차이 인원은 가장 높은 비율의 선택지에 자동 배분**
- 합계가 맞지 않으면 제출 버튼 비활성 + 인라인 경고 표시

**화면 표시 예시 (안건당):**

```
[ 안건 1. 정관 개정안 승인의 건 ] — 의장 귀속: 10명

입력 모드: [명수] / [%]

  선택지       비율(%)     → 명수
  ─────────────────────────────────
  찬성         [ 60 ]%    →  6명
  반대          [ 30 ]%    →  3명
  기권          [ 10 ]%    →  1명
  ─────────────────────────────────
  합계         100% ✅ → 10명 (불일치 시 문구노출: 🔴 100% 필요)

[ 투표 행사 ]
```

> **⚠️ 확인 필요**  
> 의장 귀속 위임 인원을 선택지별로 나눌 때, 각 위임자를 **어떤 선택지에 배정할지는 시스템이 임의로 결정**합니다.  
> (예: 찬성 6명 배정 시 명단 상위 6명 → 찬성으로 기록)  
> 이 방식이 의결 효력상 문제가 없는지 확인 부탁드립니다.

> **개발 적용 범위**  
> - 작업 규모: 🟡 중  
> - 수정 파일: `bulk-vote.php` (선택지별 수량 입력 UI로 전면 변경) / `bulk-vote_action.php` (분배 로직 변경)
