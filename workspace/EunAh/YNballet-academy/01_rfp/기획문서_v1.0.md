## 3. 과정 체계 정의

과정은 크게 **3개 카테고리**로 구분된다.

---

### 카테고리 1 — 소매틱 + 발레 스트레칭

**대상:** 20대~70대 전 연령 성인
**특징:** 소매틱 움직임 + 발레 스트레칭 결합. 유연성·바른 자세 목적.
**수업 형태:** 매트 & 센터

| 과정코드 | 과정명 | 수업 구성 | 수강 조건 |
|----------|--------|-----------|-----------|
| ST-00 | 소매틱+발레 스트레칭 | 매트 · 센터 | 전 연령 성인, 조건 없음 |

---

### 카테고리 2 — 성인 발레 (레벨별)

**대상:** 성인 취미반
**특징:** 경험·실력 기준 레벨 4단계 구분

| 과정코드 | 과정명 | 레벨 | 수강 조건 | 수업 구성 |
|----------|--------|------|-----------|-----------|
| BL-00 | 입문반 | 입문 | 발레 첫 시작 | 매트 · 손 허리바 · 기초 센터 |
| BL-05 | 기초반 | Lv. 0.5 | 기초 3개월 이상 | 매트 · 한 손바 · 센터 |
| BL-10 | 초급반 | Lv. 1 | 배운 경험 6개월 이상 | 매트 · 한 손바 · 센터(기본+테크닉) · 몸 방향 시작 |
| BL-15 | 중급반 | Lv. 1.5 | 배운 경험 1년 이상 | 한 손바 · 센터 |

---

### 카테고리 3 — 기타 과정

| 과정코드 | 과정명 | 대상 | 비고 |
|----------|--------|------|------|
| PR-01 | 개인레슨 | 성인 전 레벨 | 별도 상담 후 일정 조율 |
| YT-01 | 유아반 | 유아 | 상세 내용 추가 필요 |
| YT-02 | 청소년반 | 청소년 | 상세 내용 추가 필요 |

> ⚠️ **확인 필요:** 유아반·청소년반 수강 조건(연령 기준), 수업 구성, 개인레슨 요금 정책은 별도 확인 후 DB 입력 필요.

## 6. 문의 상태 정책

| 상태값 | 표시 | 색상 | 전환 조건 |
|--------|------|------|-----------|
| 0 | 미답변 | 빨강 | 초기 등록 시 기본값 |
| 1 | 답변완료 | 초록 | 관리자 답변 저장 시 자동 전환. **수동 변경도 가능** |
| 2 | 보류 | 노랑 | 관리자 수동 선택 |

**상태 전환 규칙:**

모든 상태 간 전환은 관리자가 자유롭게 수동 변경 가능하다. 답변완료 상태도 미답변 또는 보류로 되돌릴 수 있다. 단, 답변 텍스트가 입력된 상태에서 미답변으로 변경 시 기존 답변 내용은 유지된다 (삭제되지 않음).

```
미답변 ⇄ 보류 ⇄ 답변완료   (모든 방향 전환 가능)
```

## 7. DB 설계 (확정)

```sql
-- 공지사항
CREATE TABLE notice (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  title       VARCHAR(255) NOT NULL,
  content     TEXT NOT NULL,             -- Quill.js HTML 저장
  thumbnail   VARCHAR(500),              -- 첫 번째 img src 자동 추출
  is_active   TINYINT DEFAULT 1,         -- 1=노출, 0=숨김
  created_at  DATETIME DEFAULT NOW(),
  updated_at  DATETIME ON UPDATE NOW()
);

-- 과정소개
CREATE TABLE course (
  id          INT AUTO_INCREMENT PRIMARY KEY,
  category    VARCHAR(50) NOT NULL,      -- somatic / adult_ballet / etc
  title       VARCHAR(255) NOT NULL,     -- 과정명
  level       VARCHAR(50),               -- 입문 / Lv.0.5 / Lv.1 / Lv.1.5
  target      VARCHAR(255),              -- 수강 조건
  composition TEXT,                      -- 수업 구성 (매트·한손바·센터 등)
  fee         VARCHAR(100),              -- 수강료
  description TEXT,                      -- 과정 상세 설명
  sort_order  INT DEFAULT 0,             -- 카테고리 내 노출 순서
  is_active   TINYINT DEFAULT 1,
  created_at  DATETIME DEFAULT NOW(),
  updated_at  DATETIME ON UPDATE NOW()
);

-- 문의게시판
CREATE TABLE inquiry (
  id           INT AUTO_INCREMENT PRIMARY KEY,
  inq_type     VARCHAR(50) NOT NULL,     -- 수강문의/등록문의/기타
  course_name  VARCHAR(255),             -- 관심 과정명 (선택)
  name         VARCHAR(100) NOT NULL,
  phone        VARCHAR(20) NOT NULL,
  email        VARCHAR(150),
  title        VARCHAR(255) NOT NULL,
  content      TEXT NOT NULL,
  password     VARCHAR(255) NOT NULL,    -- password_hash() 저장
  answer       TEXT,                     -- NULL = 미답변 (상태 변경해도 유지)
  status       TINYINT DEFAULT 0,        -- 0=미답변 1=답변완료 2=보류
  created_at   DATETIME DEFAULT NOW(),
  answered_at  DATETIME                  -- 최초 답변 등록 시각
);
```

## 8. Quill.js 이미지 처리 방식

**채택 방식:** 서버 업로드 (Base64 방식 미사용)

```
이미지 삽입
    ↓
Custom Image Handler (JS)
    ↓
AJAX → /admin/upload_image.php
    ↓
/uploads/ 폴더 저장
    ↓
반환된 URL을 img src에 삽입
    ↓
저장 시 content에서 첫 번째 img src 추출 → thumbnail 컬럼 저장
```

```php
// 썸네일 자동 추출 (저장 시 처리)
preg_match('/<img[^>]+src=["\']([^"\']+)["\']/', $content, $matches);
$thumbnail = $matches[1] ?? null;
```

**이미지 업로드 보안 처리:**
허용 확장자는 jpg, jpeg, png, gif, webp만 허용하고 MIME 타입을 병행 검증한다. 업로드 파일명은 `uniqid()` 기반으로 랜덤 변환하여 저장한다.

## 9. 보안 처리 정책

| 항목 | 처리 방식 |
|------|-----------|
| 관리자 인증 | 세션 기반. 미로그인 시 `/admin` 전체 접근 차단 → 로그인 리다이렉트 |
| 관리자 계정 | ID/PW 하드코딩. PW는 `password_hash()` 적용값으로 코드에 저장 |
| 문의 비밀번호 | `password_hash()` 저장 / `password_verify()` 검증 |
| 이미지 업로드 | 확장자 + MIME 타입 화이트리스트 검증. 파일명 랜덤화 |
| XSS 방지 | 출력 시 `htmlspecialchars()` 적용. 공지 본문(Quill HTML)은 허용 태그 필터링 별도 처리 |
| SQL Injection | `mysqli_prepare()` prepared statement 전면 적용 |

## 10. 개발 우선순위

**1단계 — 기반**
DB 생성 + `db.php` + 관리자 로그인/세션 + 이미지 업로드 엔드포인트

**2단계 — 관리자 기능**
공지 CRUD(Quill.js) → 과정 CRUD → 문의 목록/답변/상태 변경

**3단계 — 사용자 화면**
랜딩 + 소개(정적) + 과정소개(DB 연동, 탭) + 공지 목록/상세 + 문의 작성/목록/상세

**4단계 — 마무리**
Bootstrap 반응형 점검 + 보안 전체 검토 + 지도 이미지 삽입 + 브라우저 크로스체크
