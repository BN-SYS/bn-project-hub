## 2. 공통 요구사항

### 2-1. 레이아웃
- 전 페이지 Bootstrap 반응형 적용 (모바일·태블릿·데스크탑)
- 공통 헤더: 로고 + GNB (랜딩 / 소개 / 과정 / 공지 / 문의)
- 공통 푸터: 아카데미명, 주소, 연락처, 저작권

### 2-2. 보안
- 모든 DB 조회·저장: `mysqli_prepare()` prepared statement
- 사용자 입력 출력 시: `htmlspecialchars()` 전면 적용
- 공지 본문(Quill HTML) 출력: 허용 태그만 렌더링 (img, p, br, strong, em, ul, ol, li, a)
- 관리자 전 페이지: 세션 유효성 검사 → 미로그인 시 `/admin/login.php` 리다이렉트

### 2-3. 임시 데이터 처리
- 이미지 미정 항목: `/assets/img/placeholder_*.jpg` 로 표기
- 텍스트 미정 항목: `[임시] 내용 준비 중입니다.` 로 표기
- 히어로 슬라이드 문구: `[임시] 몸이 기억하는 아름다움` (확정 슬로건 적용, 서브문구는 임시)
- 강사 소개 사진·경력: placeholder 이미지 + `[임시]` 텍스트

### P01. 랜딩페이지 (`/index.php`)

**목적:** 첫 방문자가 아카데미를 파악하고 수강 문의로 유도

#### 섹션 구성

**[섹션 1] 히어로 배너**
- Bootstrap Carousel, 슬라이드 2개
- 각 슬라이드: 배경 이미지 + 오버레이 + 문구(제목+서브) + CTA 버튼
- 슬라이드 1: `[임시]` 이미지, 제목: "몸이 기억하는 아름다움", 서브: `[임시]`, CTA: "과정 보기" → `/course.php`
- 슬라이드 2: `[임시]` 이미지, `[임시]` 문구, CTA: "수강 문의하기" → `/inquiry/write.php`
- 자동 전환 5초, 인디케이터·화살표 표시

**[섹션 2] 아카데미 한줄 소개**
- 정적 텍스트 블록. 문구: `[임시] 내용 준비 중입니다.`

**[섹션 3] 과정 미리보기**
- DB: `course` 테이블, `is_active=1`, `sort_order` ASC, 최대 4개
- 카드 형태: 카테고리 배지 + 과정명 + 수강 조건 한줄
- "전체 과정 보기" 버튼 → `/course.php`

**[섹션 4] 공지사항**
- DB: `notice` 테이블, `is_active=1`, `created_at` DESC, 5건
- 출력: 썸네일(60×60) + 제목 + 날짜
- 썸네일 없으면 기본 이미지(`/assets/img/notice_default.jpg`) 대체
- 제목 클릭 → `/notice/view.php?id={id}`
- "공지 전체보기" 링크 → `/notice/list.php`

**[섹션 5] 문의 CTA**
- 배경 컬러 블록, 문구 + "수강 문의하기" 버튼 → `/inquiry/write.php`

### P03. 과정소개 (`/course.php`)

**목적:** 과정 상세 안내 및 문의 유도

#### 카테고리 탭

| 탭 | category 값 | 표시명 |
|----|------------|--------|
| 1 | `somatic` | 소매틱+발레 스트레칭 |
| 2 | `adult_ballet` | 성인 발레 |
| 3 | `etc` | 기타 과정 |

- 기본 활성 탭: 첫 번째 탭 (somatic)
- URL 파라미터 `?tab=` 로 직접 진입 가능
- 탭 전환: JS (페이지 리로드 없이 Bootstrap tab 또는 JS toggle)

#### 과정 카드

- DB: `course`, `category = 선택탭`, `is_active=1`, `sort_order` ASC
- 카드 출력 항목:
  - 과정명 (title)
  - 레벨 (level) — 없으면 미표시
  - 수강 조건 (target)
  - 수업 구성 (composition)
  - 수강료 (fee) — `etc` 카테고리의 `PR-01`(개인레슨)은 "문의 후 안내" 고정 표시
- 유아반·청소년반: 데이터 미정 → `[임시] 내용 준비 중입니다.` 표기

#### 하단

- 등록 안내: 정적 텍스트 (준비물, 등록 절차) — `[임시]` 처리
- "수강 문의하기" CTA 버튼 → `/inquiry/write.php`

#### 입력 폼

| 필드 | 타입 | 필수 | 검증 규칙 |
|------|------|------|----------|
| 이름 | text | ✅ | 1자 이상 |

| 이메일 | email | ❌ | 형식 검증 (입력 시) |
| 문의 유형 | select | ✅ | 수강문의 / 등록문의 / 기타 |

| 내용 | textarea | ✅ | 10자 이상 |
| 비밀번호 | password | ✅ | 4~8자리 숫자 |
| 개인정보 동의 | checkbox | ✅ | 미체크 시 제출 차단 |

- 제출 성공: 문의 목록 페이지로 리다이렉트 + 완료 메시지

### P08. 문의 상세 (`/inquiry/view.php?id={}`)

#### 접근 제어

```
요청 진입
  ↓
관리자 세션? → YES → 바로 출력
  ↓ NO
비밀번호 입력 폼 출력
  ↓
password_verify() → 일치 → 출력
             → 불일치 → "비밀번호가 틀렸습니다" + 재입력 유도
```

#### 출력 항목

- 문의 유형 · 관심 과정 · 이름 · 연락처 · 이메일 · 제목 · 내용 · 등록일
- 관리자 답변 영역: 답변 있으면 출력, 없으면 "답변 준비 중입니다."

## 4. 관리자 화면 요구사항

### 공통

### A01. 관리자 로그인 (`/admin/login.php`)

- ID/PW 하드코딩 (`config.php` 별도 분리)
- PW: `password_hash()` 적용값 저장
- 로그인 성공: `$_SESSION['admin'] = true` → `/admin/notice_list.php`
- 실패: "아이디 또는 비밀번호가 틀렸습니다" (성공/실패 구분 메시지 X)
- 이미 로그인 상태: 공지 목록 리다이렉트

### A09. 문의 상세/답변 (`/admin/inquiry_view.php?id={}`)

**출력:** 문의 전체 정보 (유형·과정·이름·연락처·이메일·제목·내용·등록일)

**답변 영역:**
- textarea (기존 답변 있으면 로드)
- "답변 저장" 버튼: answer 컬럼 저장 + `answered_at` 최초 1회만 기록

**상태 변경:**
- select: 미답변(0) / 답변완료(1) / 보류(2)
- 답변 저장 시 status 자동 1 전환 (이미 1·2면 그대로 유지)
- select로 수동 변경 별도 가능 (모든 방향)
- 답변 삭제 기능 없음. answer 텍스트 있어도 status 0 복귀 가능

**하단:** "목록으로" → `/admin/inquiry_list.php`

## 5. 이미지 업로드 요구사항 (`/admin/upload_image.php`)
